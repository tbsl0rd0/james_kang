james_kang 깃허브
The james_kang project is a website introducing myself you are currently viewing.  It is a project for introducing myself and my own projects and philosophy that I formed while having developed.

Because of a nature of the project, a static web is sufficient for this project and it does not have to be exposed to search engines by resource, so it routes resources using Angular route module.  The static web server is combination of AWS Route53 and S3.

웹팩을 이용한 컴포넌트 번들화
Component bundling with webpack
웹팩의 로더를 통해서 위의 컴포넌트 구조의 js 파일에서 html 파일을 스트링으로 읽어서 템플릿으로 지정하고 less 파일을 컴파일하여 css로 지정합니다.  이렇게 1차적으로 번들화된 컴포넌트 파일을 묶어 최종적인 번들 파일을 만들어 냅니다.

--watch 옵션을 통해 이런 일련의 작업들은 코드가 변경될 때마다 자동으로 실행됩니다.
Through the webpack loader, it reads the html file as a string and designate it as a template, and compile the less file and load it into css from the js file of the component structure above.  These primary bundled component files are bundled together to produce the final bundle file.

With the --watch option, this set of tasks is automatically executed whenever the code changes.
걸프를 이용한 번들 파일과 라이브러리의 통합
Integration of bundle files and libraries using gulf
걸프는 웹팩을 통해서 만들어진 번들 파일과 필요한 라이브러리들을 한데 묶어 minify를 하고 하나의 파일로 통합합니다.  최종적으로 자바스크립트와 관련된 파일은 javascript.min.js로, CSS와 관련된 파일은 css.min.css로 만들어집니다.  이렇게 하나의 파일로 통합했을 때는 요청을 여러번 하는 대신 하나의 파일만 요청하게 하므로서 과부하를 막을 수 있고 관리의 편의성이 있습니다.

걸프도 마찬가지로 코드가 변경될 때마다 watch 태스크를 통해서 자동으로 실행됩니다.
Gulf bundles bundle file made with webpack and necessary libraries, minifies and integrates it into one file.  Finally, the javascript-related files are created as javascript.min.js, and the CSS-related files are created as css.min.css.  When combined into a single file, it can prevent overloading by making only one file to be requested instead of multiple requests.

The gulf also runs automatically through the watch task whenever the code changes.

앵귤러 라우트 모듈을 이용한 라우팅
Routing with an angular route module
정적 웹 서버 환경에서 URL 라우팅을 하려면 HTML 문서의 상대적 위치에 따라서 라우팅을 하기 때문에 결국 정적 사이트 제너레이터나 여러 개의 HTML 문서를 만들어야 합니다.  하지만 검색 엔진에 이러한 리소스별 노출이 불필요하다면 앵귤러 라우트 모듈을 이용하여 컴포넌트 단위로 라우팅해도 무방해 보입니다.  james_kang 프로젝트는 블로그와는 달리 자원별 노출이 불필요 합니다.  따라서 다음과 같은 #!을 포함하는 URL을 이용하여 컴포넌트 단위로 리소스를 라우팅합니다.
In a static web server environment, URL routing is performed based on a relative location of the HTML document, so it needs to create a static site generator or multiple HTML documents.  However, if it does not need these resource-specific exposures to search engines, it might be able to route them on a component-by-component basis using the angular route module.  The james_kang project does not require exposure by resource, unlike a blog.  Therefore, resources are routed on a component basis using the URL that contains #!

정적 웹 서버
Static web server
james_kang 프로젝트는 동적 요소가 없어서 정적 웹 서버에서 서브합니다.  정적 웹 서버는 AWS의 라우트53과 S3를 이용했습니다.  라우트53은 단순히 james-kang.me 도메인과 S3 리소스들을 연결합니다.  S3는 AWS 고유의 파일 서버이고 S3에 index.html과 웹팩과 걸프가 생성한 파일들과 이미지 파일들을 올립니다.
The james_kang project has no dynamic elements and serves on a static web server.  The static web server is combination of AWS route53 and S3.  Route53 simply connects the james-kang.me domain with the S3 resources.  S3 is AWS's own file server and uploads index.html and files generated by webpack and gulf and image files.
