Philosophy - Naming
표기법
Notation
표기법은 언더바 표기법을 사용합니다.
I use underbar notation.
카멜 표기법 : appleTree
하이픈 표기법 : apple-tree
언더바 표기법 : apple_tree
Camel notation : appleTree
Hyphen notation : apple-tree
Underbar notation : apple_tree
카멜 표기법은 타이핑하기도 어렵고 막상 적어놓고 보면 한눈에 단어를 알아보기가 쉽지 않습니다.  또한, 하이픈 표기법은 자바스크립트에서 허용하지 않고 더블클릭했을 때 한번에 선택되지 않습니다.  언더바 표기법은 타이핑하기도 쉽고 가독성도 좋으며 더블클릭했을 때 한번에 선택되므로 코딩속도가 향상됩니다.

그러나 다음과 같이 언더바 표기법이 허용되지 않는 경우에는 카멜 표기법이나 하이픈 표기법을 사용합니다.
It is not easy to type in camel notation and to know that at a glance.  Hyphen notation is not allowed in JavaScript and when double clicked, it is not blocked as a word.  But, as well as underbar notation is easy to type, it is blocked at once, so it is possible to speed up the coding.

Yet, when it is not allowed as follows, there is no choice to use the other two.
HTML 사용자 정의 태그 이름 : 하이픈 표기법
앵귤러 컴포넌트 이름 : 카멜 표기법
도메인 네임 : 하이픈 표기법
HTML custom tag name : hyphen notation
Angular component name : camel notation
Domain name : hyphen notation
약어
Abbreviation
코딩할 때 자주 등장하고 풀네임으로 타이핑하기 번거로운 7글자 이상의 다음과 같은 단어에 대해서 약어를 사용합니다.
I use the abbreviation for the following words that appear frequently when it comes to coding and the words that are hard to type as full name and seven characters or more.
initialize    init
language    lang
request    req
response    res
document    doc
