발단
Prolog
PC를 사용하는 거의 대부분의 사람들은 윈도우 운영체제를 사용합니다.  이는 여러 회사에서도 마찬가지일 것입니다.  특히 회사에서는 윈도우 PC에 로그인할 때 어느정도 보안성이 필요하다고 생각합니다.

그러나 윈도우 로그인 암호가 공유되거나 편의상의 이유로 매우 쉬운 암호로 설정하는 경우가 많습니다.  그래서 패스워드 이외에 윈도우에 로그인하는 다른 인증 방법이 필요하다고 생각했습니다.  그리고 간소한 인증 방법부터 번거롭지만 보안성이 높은 인증 방법까지 다양하게 필요하다고 생각했습니다.  간소한 인증 방법은 지식기반의 인증 방법이 아니라 소유기반의 인증 방법이 적절하다고 생각합니다.  지식기반의 인증 방법은 한번 공유되면 계속 공유되는 특징 때문에 보안성이 상당히 떨어지기 때문입니다.

이렇게 되면 PC에서는 인증 방법은 간소화되면서 어느정도 보안성도 높힐 수 있습니다.  그러면서 서버 컴퓨터같이 보안이 중요한 컴퓨터는 다중인증을 통해 보안성을 매우 높힐 수 있습니다.

smart_auth 프로젝트는 회사에서 사원들이 사용하는 것을 모델로 시작되었고 세 명이 힘을 모아 분업, 협업하면서 완성한 프로젝트입니다.

smart_auth 깃허브
Almost everyone who uses a PC uses the Windows operating system.  This will be the same for many companies.  In particular in company, I think users need some security when logging in to a Windows PC.

However, it is often the case that the Windows login password is shared or set to a very easy password for convenience.  So I thought it needs another authentication method to login to Windows in addition to the password.  And I thought it needs various ways from simple authentication method to cumbersome but highly secure authentication method.  I believe that a simple authentication method should be not a knowledge-based authentication method, but an ownership-based authentication method.  Knowledge-based authentication methods have the feature that once shared, continue to share that significantly reduce security.

This simplifies the authentication method on the PC and can also increase the security to some extent.  However, security-conscious computers, such as server computers, can greatly enhance security through multiple authentication.

The smart_auth project is a project that began as a model of what employees use in the company, and is a project that has been completed by three people working together as partisans and collaborators.

사원증
Smart ID card
사원증이란 위의 구성도에서 Smart ID card를 의미합니다.  PC에 아두이노가 연결되어 있고 이 아두이노에 BLE 모듈이 마스터 모드로 주위의 BLE를 스캔하는 옵저버 역할을 합니다.  사원증 내에 삽입되어 있는 BLE는 슬레이브 모드로 작동하고 비콘의 역할을 수행합니다.
The arduino is connected to the PC and the BLE module in this arduino acts as an observer in master mode to scan the surrounding BLE.  The BLE inserted in the Smart ID card operates in slave mode and acts as a beacon.
구글 OTP
Google OTP
구글 OTP는 구글에서 제공하는 시간기반의 OTP 클라이언트입니다.  이 구글 OTP 클라이언트와 연동시켜 윈도우 로그인에 OTP를 사용합니다.
Google OTP is a time-based OTP client provided by google.  It uses OTP for Windows login in conjunction with this Google OTP client.
동글인
Donglein
동글인은 동글과 로그인의 합성어로 새로이 만든 인증 방법입니다.  동글처럼 로그인한다고 해서 만들어진 이름입니다.  동글인에서 사용하는 동글은 일반 USB로서 MBR 영역에 인증키를 숨겨 안전하게 보관하는 방법을 사용합니다.
A dongle is a combination of dongle and login.  It is a name created by logging in like a dongle.  The dongle used in the donglein is a generic USB and uses a method of safely keeping the authentication key in the MBR area.
크리덴셜 프로바이더
Credential provider
크리덴셜 프로바이더는 윈도우에서 사용하는 자격증명(크리덴셜)을 공급하기 위한 DLL입니다.  크리덴셜 프로바이더를 규격에 따라 제작하고 윈도우 레지스트리에 등록하면 윈도우 로그인 화면이 뜰 때 기본 크리덴셜 프로바이더를 대체하여 로드됩니다.  이 크리덴셜 프로바이더의 로직에 따라서 인증을 수행하고 윈도우에 로그인할 수 있게 합니다.
A credential provider is a DLL for supplying the credentials used by Windows.  Creating a credential provider and register it in the Windows registry, it will be loaded in place of the default credential provider when the Windows login screen appears.  Depending on the logic of this credential provider, it can perform authentication and login to Windows.
